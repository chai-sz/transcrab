## 一、GPT-5.6 有哪些变化

简要概述 GPT-5.6 的根本性变化——在你动笔写任何提示词之前，这些背景很重要。

1. **Token 效率大幅提升**

GPT-5.6 在达到前沿性能的同时，比之前的模型使用更少的输出 token。

2. **意图理解更好**

GPT-5.6 能够推断底层目标和适当的执行力度，无需你一步步说清楚。你仍然需要明确说明你的约束、审批边界和成功标准，但模型不需要那么多手把手指导就能到达目标。

3. **回复默认更短**

这是最大的行为变化之一。GPT-5.6 的回复全面比之前的模型更短：更少的通用开场白、更少的推测分支、更短的列表，以及答案与推理之间更少的重复。

4. **设计和前端判断力提升**

GPT-5.6 能生成更精良、更可用的网站和应用，在布局、视觉层次和设计判断上明显更强。

5. **新的推理与执行模式**

GPT-5.6 为最困难的任务引入了新的最大推理力度（超越 xhigh），以及全新的 Pro 模式（后面详述）。

6. **工具调用获得新的执行路径**

GPT-5.6 可以编写并运行自己的轻量级程序来调用工具、处理结果和过滤中间输出，称为程序化工具调用（Programmatic Tool Calling）。

7. **多智能体协调（Beta）**

单个 GPT-5.6 实例现在可以并行协调多个子智能体并综合它们的结果，类似于 Codex 中的 ultra 模式。

## 二、如何正确编写 GPT-5.6 提示词

1. **使用更短的提示词**

这是你需要做的最大的调整。

在 OpenAI 的内部评估中，他们发现将冗长、明确的系统提示替换为最小提示后，分数提升了约 15%，同时总 token 减少了 60% 以上。

2. **明确界定自主权和权限**

GPT-5.6 可以主动且持续地行动，这意味着你需要为每个请求定义允许采取的行动级别。

使用 GPT-5.6 时，只要清楚告诉模型在哪里停下来，以及它实际被授权做什么就行。

一个重要的注意事项：不要在你的提示词中反复说"先问一下"、"不要修改"或"等待批准"。重复可能会触发不必要的权限检查，即使是对于安全的、预期的操作。

3. **避免泛泛的指令**

使用优先级指令——这告诉 GPT-5.6 你的底层目标。

示例：如果你想要更短的回复，不要说"简短点"，而要说"先说结论"。

4. **语气**

告诉 GPT-5.6"更友好"或"更有同理心"，不会带来有意义的改进。

通用的温暖指令不再像以前模型那样起作用了。

5. **保持结构轻量**

GPT-5.6 受益于少量的任务特定结构——一个轻量级的大纲，而不是在每个回答上都强加完整的回复模板。

只在你自己的测试证明确实需要的情况下，才添加狭窄的约束。

TLDR：对 GPT-5.6 来说，少真的就是多。更短的提示词、更清晰的权限边界取代重复警告、具体的指令取代模糊的指令。过度工程化你的提示词现在更可能有害而不是有益。

## 三、新功能

GPT-5.6 附带的所有新功能概览

1. **Pro 模式**
Pro 模式可以与任何 GPT-5.6 模型（Sol、Terra 或 Luna）及其支持的任意推理力度级别配合使用。

你不需要切换到单独的"Pro"模型；只需保留现有的模型选择，在其上启用 Pro 模式设置即可。

2. **ChatGPT Work**

ChatGPT Work 是 OpenAI 对 Claude Cowork 的直接竞品。

工作模式与常规 ChatGPT 聊天分开，旨在自动化你行业中的真实工作流。

3. **GPT-Live**

OpenAI 新的高级语音提示功能。

更流畅的对话、实时语言翻译等。

4. **新的推理力度**

GPT-5.6 现在支持六种推理力度级别：

None、Low、Medium、High、xHigh 和 Max。新增的是 Max，位于 xhigh 之上，针对最复杂的任务。

你可以在 ChatGPT 的模型选择器中直接更改推理力度。

5. **Hosted Sites**

一种分享创意的新方式。

Sites 通过网页版 ChatGPT Work 访问，或通过 ChatGPT 桌面应用的 Work/Codex 模式访问。

## 四、GPT-5.6 最佳提示词框架

结合以上所有内容的精确框架，适用于大多数提示词：

**第 1 步：选择模型**

GPT-5.6 有三个层级：

- Sol：用于最困难的任务
- Terra：日常使用
- Luna：最快、最经济——专为高吞吐量工作负载而设计

**第 2 步：设置推理力度**

None/Low：基础提示词

Medium：默认（用于涉及 MCP 调用、API 调用等的提示词）

High 或 xHigh：复杂任务

Max：最后手段（用于极其复杂的工作）

**第 3 步：决定是否需要 Pro 模式**

与推理力度分开。

只有在质量上的边际提升真正重要时才使用。

**第 4 步：构建提示词**

每个高质量的 GPT-5.6 提示词都有四个组成部分：

1. **结果**：完成到底是什么样的
2. **约束**：审批边界、绝对不能发生的事情、范围
3. **证据/成功标准**：什么证明或格式可以确认它是正确的
4. **自主权策略**：模型可以在不询问的情况下自行做什么

## 五、GPT-5.6 的注意事项

- 缓存写入现在更贵了。按未缓存输入价格的 1.25 倍计费。
- 程序化工具调用并不总合适。当一次调用就足够时跳过它。
- Pro 模式成本更高。所有额外的后台工作按标准 token 费率计费。
- Sites 的部署 URL 始终是生产环境。默认没有单独的预发布步骤。如果想在上线前审核，请先保存一个版本。
- GPT-5.4 将于 7 月 23 日退役。GPT-5.5 仍然可用。最好现在就迁移，而不是等待。
- 旧的 GPT-5.5 提示词可能无法直接迁移。让 GPT-5.6 重写你的工作流，使新的 5.6 模型能够更高效地工作。
